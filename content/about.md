안녕하세요. 하드웨어와 운영체제의 경계에서 동작하는 소프트웨어를 공부하는 **harunohi**입니다.

이 블로그에는 결과만 나열하지 않고, 문제를 재현하고 원인을 좁혀 해결하기까지의 과정을 기록합니다. 학습 기록이면서 동시에 제가 어떤 방식으로 문제를 이해하고 해결하는지 보여주는 포트폴리오입니다.

## 관심 분야

- C/C++ 기반 임베디드 소프트웨어
- Linux Kernel과 디바이스 드라이버
- MCU 펌웨어 분석과 디버깅
- Raspberry Pi와 Embedded Linux
- 바이너리 분석과 리버스 엔지니어링
- 재현 가능한 Linux 개발 환경

## 기록 원칙

1. 실행 환경과 재현 조건을 명확히 남깁니다.
2. 시행착오와 기술적 선택의 이유를 함께 기록합니다.
3. 코드와 측정 결과를 통해 결론을 검증합니다.
4. 프로젝트 결과는 GitHub 저장소와 관련 글로 연결합니다.
